# Roadmap

## Phase 0 — Foundation (current)
- Create a `docs/` canonical-document set.
- Establish product vision, product state, and near-term roadmap.
- Capture a baseline reference on skill capabilities (hooks, scripts, references, assets) from primary ecosystem docs.

## Phase 1 — Canonical docs framework
- Add document templates and quality bars for project context capture.
- Define update cadence and ownership for canonical docs.
- Add a lightweight validation checklist to keep docs current and actionable.

## Phase 2 — Skill schema and repository conventions
- Define recommended folder conventions for skill bundles in this repo.
- Add examples showing how project docs map into skill `references/` and operational scripts.
- Publish guidance for interoperability across agent implementations.

## Phase 3 — Tooling and automation
- Add scripts for linting/validating skill metadata and doc completeness.
- Add helper automation for creating new project-context skill packs.
- Add CI checks to prevent drift between canonical docs and skill artifacts.

## Phase 4 — Reference implementations
- Provide at least one fully worked skill package for a realistic software project.
- Include end-to-end examples demonstrating hooks + scripts + references + assets collaboration.
- Document adaptation patterns for different agent products.

## Phase 5 — Evaluation and iteration
- Define success metrics (adoption, task success, context retention, onboarding speed).
- Run pilot usage and capture failure modes.
- Iterate docs and structure based on observed agent behavior.
