# Product Vision

## Vision statement
Build the canonical, interoperable home for **agent skills**: a project-first system where teams can capture operational knowledge as reusable skills, preserve project context over time, and distribute those skills across agent platforms without rewriting them.

## Problem we are solving
Teams repeatedly lose context in AI-assisted development and operations:
- Important project conventions live in chat history instead of durable docs.
- Agent behavior is inconsistent across tools and sessions.
- Procedural know-how is duplicated as ad-hoc prompts rather than reusable artifacts.

## Product direction
This repository will define and model:
1. **Canonical project documents** that describe intent, constraints, and current state.
2. **Skill-oriented context packaging** so instructions, references, scripts, and assets are versioned together.
3. **Cross-platform alignment** with emerging Agent Skills patterns used by major agent ecosystems.

## Design principles
- **Canonical over conversational**: key project truth should live in version-controlled docs.
- **Progressive disclosure**: keep top-level instructions lean; load references/scripts/assets when needed.
- **Portable by default**: structure documentation so it maps cleanly to multiple agent runtimes.
- **Operationally useful**: every document should help an agent (or human) execute real work.

## Target outcomes
- Faster agent onboarding to any project using this kit.
- Reduced prompt repetition through reusable skills and references.
- Higher consistency in agent outputs across tasks and tools.
- Better auditability of project context changes over time.
